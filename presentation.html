CivicSight AI:
Citizen Reporting + City Operations

A production-style system combining the Flutter mobile app and admin portal to capture civic issues, prioritize them with AI, assign field workers, and close the loop with transparent status updates.

Mobile: Citizen + Worker Admin Portal Supabase + Firebase Auth AI Triage Pipeline
Problem

Municipal issue reporting is fragmented and slow

Citizen Friction

Users struggle with disconnected forms, low confidence in response times, and poor visibility into progress.

Operational Bottlenecks

Manual triage and unclear severity classification cause delayed assignments and inconsistent service levels.

No clear ownership Inconsistent prioritization Limited accountability

Worker Portal

Admin Portal

Review reports, assign workers, supervise task flow, and monitor lifecycle states across active incidents.

Report Management

Architecture

End-to-end stack powering mobile + admin

Frontend

Flutter mobile app for citizens/workers and a React/Next-style admin interface for city operations.

Backend

Supabase Postgres, APIs, and secure auth integration for data consistency and role-based access.

Delivery

Branch-based workflow with review-first PRs for safer merges across mobile and admin codebases.
